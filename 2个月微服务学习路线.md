### Week 8: 面试准备

#### Day 1-3: 八股文准备

**Spring Boot 高频问题**：

1. **Spring Boot 自动配置原理？**
   - `@SpringBootApplication` = `@Configuration` + `@EnableAutoConfiguration` + `@ComponentScan`
   - `@EnableAutoConfiguration` 通过 `SpringFactoriesLoader` 加载 `META-INF/spring.factories`
   - 根据条件注解（`@ConditionalOnClass`）决定是否生效

2. **Bean 的生命周期？**
   - 实例化（Instantiation）
   - 属性赋值（Populate）
   - 初始化（Initialization）
     - `@PostConstruct`
     - `InitializingBean.afterPropertiesSet()`
     - `init-method`
   - 销毁（Destruction）
     - `@PreDestroy`
     - `DisposableBean.destroy()`
     - `destroy-method`

3. **`@Autowired` vs `@Resource`？**
   - `@Autowired`：Spring 注解，按类型注入，可配合 `@Qualifier` 按名称
   - `@Resource`：JDK 注解，默认按名称注入，找不到再按类型

4. **AOP 的实现原理？**
   - JDK 动态代理（接口）：基于反射，生成接口的代理类
   - CGLIB 动态代理（类）：基于字节码，生成子类

5. **事务失效的场景？**
   - 方法不是 public
   - 同类内部调用（没走代理）
   - 异常被 catch 了
   - 异常类型不对（默认只回滚 RuntimeException）
   - 数据库不支持事务（MyISAM）

---

**Spring Cloud 高频问题**：

1. **服务注册与发现的原理？**
   - 服务启动时向注册中心注册（IP、端口、服务名）
   - 定时发送心跳（默认30秒）
   - 消费者从注册中心拉取服务列表（定时更新）
   - 通过负载均衡选择一个实例调用

2. **Feign 的工作原理？**
   - 启动时扫描 `@FeignClient` 注解
   - 为接口生成动态代理
   - 调用时将方法参数转为 HTTP 请求
   - 通过 LoadBalancer 选择服务实例
   - 发送 HTTP 请求并解析响应

3. **熔断降级的区别？**
   - **熔断**：当错误率达到阈值，直接拒绝请求（快速失败）
   - **降级**：服务不可用时，返回默认值或调用备用方法
   - 熔断是自动的，降级可以手动触发

4. **网关的作用？**
   - 统一入口
   - 路由转发
   - 鉴权认证
   - 限流熔断
   - 日志记录
   - 跨域处理

5. **配置中心如何实现动态刷新？**
   - 客户端定时轮询配置中心（长轮询）
   - 配置变更时，服务端推送通知
   - 客户端接收通知后，重新拉取配置
   - 通过 `@RefreshScope` 刷新 Bean

---

**分布式高频问题**：

1. **CAP 理论？**
   - **C（Consistency）**：一致性，所有节点同一时间看到相同数据
   - **A（Availability）**：可用性，系统一直可用
   - **P（Partition tolerance）**：分区容错性，网络分区时系统仍能工作
   - **结论**：只能同时满足两个，分布式系统必须满足 P，所以只能在 CP 和 AP 之间选择

2. **分布式事务的解决方案？**
   - **2PC/3PC**：强一致性，性能差，不推荐
   - **TCC**：Try-Confirm-Cancel，需要业务侵入
   - **Saga**：长事务，补偿机制
   - **本地消息表**：最终一致性
   - **MQ 事务消息**：最终一致性
   - **Seata AT**：自动补偿，推荐

3. **如何保证消息不丢失？**
   - **生产者**：开启确认机制（confirm）
   - **Broker**：持久化（磁盘）
   - **消费者**：手动 ACK

4. **如何设计分布式锁？**
   - **Redis**：
     - `SET key value NX EX seconds`
     - 设置唯一标识（UUID）防止误删
     - Lua 脚本保证原子性
     - Redisson 实现（看门狗自动续期）
   - **Zookeeper**：临时顺序节点
   - **数据库**：唯一索引

5. **如何保证接口幂等性？**
   - **唯一ID**：订单号、流水号
   - **Token 机制**：先获取 token，提交时验证并删除
   - **状态机**：订单状态只能从待支付→已支付
   - **分布式锁**：同一时间只能有一个请求执行
   - **数据库唯一索引**：插入时如果重复会报错

```
- 消息队列：RocketMQ
- 缓存：Redis
- 数据库：MySQL
- 链路追踪：SkyWalking
- 容器化：Docker + Docker Compose

【系统架构】
采用微服务架构，共5个核心服务：
1. 用户服务（User Service）：用户注册、登录、信息管理
2. 商品服务（Product Service）：商品管理、查询
3. 订单服务（Order Service）：订单创建、查询、取消
4. 支付服务（Payment Service）：支付、退款
5. 库存服务（Inventory Service）：库存管理、扣减

【我的职责】
1. 负责订单服务和库存服务的开发
2. 实现分布式事务（Seata）保证订单-库存-支付的一致性
3. 使用 Redis 分布式锁解决秒杀超卖问题
4. 使用 RocketMQ 实现订单异步处理
5. 使用 Sentinel 实现接口限流和熔断降级

【技术难点】

1. 分布式事务问题
   - 问题：下单时需要扣减库存、创建订单、扣减余额，如何保证一致性？
   - 解决方案：使用 Seata AT 模式
     - 在订单服务的下单方法上添加 @GlobalTransactional
     - Seata 自动记录 undo_log
     - 如果任何一个服务失败，自动回滚
   - 效果：保证了分布式事务的一致性

2. 秒杀超卖问题
   - 问题：高并发下如何防止库存超卖？
   - 解决方案：
     - 商品库存缓存到 Redis
     - 使用 Redisson 分布式锁
     - 先扣减 Redis 库存，再异步扣减数据库库存
     - 使用 RocketMQ 削峰
   - 效果：TPS 从 500 提升到 5000

3. 缓存一致性问题
   - 问题：商品信息更新后，如何保证缓存和数据库一致？
   - 解决方案：
     - 采用 Cache Aside 模式（先更新数据库，再删除缓存）
     - 使用 Canal 监听 MySQL binlog，异步更新缓存
     - 设置缓存过期时间兜底
   - 效果：保证了最终一致性

4. 接口幂等性问题
   - 问题：用户重复提交订单怎么办？
   - 解决方案：
     - 前端：按钮置灰 + 防抖
     - 后端：Token 机制（下单前获取 token，提交时验证并删除）
     - 数据库：订单号唯一索引
   - 效果：避免了重复下单

5. 服务雪崩问题
   - 问题：某个服务故障导致整个系统不可用？
   - 解决方案：
     - Sentinel 限流：QPS 限制
     - Sentinel 熔断：错误率超过阈值自动熔断
     - Sentinel 降级：返回默认值或调用备用方法
   - 效果：保证了系统的高可用
```

### 为什么选 Spring Cloud Alibaba？

1. ✅ **国内主流**：阿里系，国内公司用得多
2. ✅ **文档友好**：中文文档完善，学习成本低
3. ✅ **生态完整**：Nacos、Sentinel、Seata 一套解决方案
4. ✅ **面试高频**：面试官更熟悉，容易沟通
5. ✅ **社区活跃**：问题容易找到答案

### Spring Cloud Alibaba vs Spring Cloud Netflix

| 组件 | Spring Cloud Alibaba | Spring Cloud Netflix | 推荐 |
|------|---------------------|---------------------|------|
| 注册中心 | Nacos | Eureka（停更） | Nacos ✅ |
| 配置中心 | Nacos Config | Config（停更） | Nacos ✅ |
| 熔断限流 | Sentinel | Hystrix（停更） | Sentinel ✅ |
| 网关 | Gateway | Zuul（停更） | Gateway ✅ |
| 服务调用 | OpenFeign | OpenFeign | OpenFeign ✅ |
| 负载均衡 | LoadBalancer | Ribbon（停更） | LoadBalancer ✅ |

**结论**：Spring Cloud Netflix 大部分组件已停更，选择 Spring Cloud Alibaba！
